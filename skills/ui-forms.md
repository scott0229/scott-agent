# 表單設計與按鈕行為規範 (UI Forms)

日常的管理與大量的資料輸入需要順暢、無雜訊、直接且能夠防護失誤的表單體驗。

## 1. 高效率無雜訊表單 (Workflow Efficiency)
- 全部的輸入欄位 (Inputs/Selects) 都必須伴隨著文字描述的標籤 (Labels)；它們通常擺放被佈局在輸入框的上方或橫列的左側 (例如 Grid 佈局)。
- **消除視覺雜訊原則**：請省略表單上標籤旁邊常見用以代表必填的紅色星號 (`*`)。在這個高等級的環境中，紅色星號會帶來強烈的焦躁與視覺干擾。若欄位確實是必填項目，在 Submit 動作觸發的後端或前端檢核時，給予清楚的獨立錯誤提示即可。

## 2. 高風險行為的前置防護 (High-Friction Destructive Actions)
- 面對如「刪除全部」、「重設」、「清除帳戶資料」等具破壞力、不可逆轉的操作，就算是在一長串名單中點擊一個垃圾桶小圖示，**也必須**使用如 `AlertDialog` 這樣具阻擋性的對話框來跳出二次確認視窗，務必預防因疲勞而產生的誤觸。
- 前往觸發這類高風險確認的按鈕設計，在非互動狀況下請保持內斂的留白外框形式 (`variant="outline"`)，但在 Hover 懸停準備點擊時才加上明顯的警告色來提示危險性 (寫作標準如：`className="hover:bg-red-50 hover:text-red-600 border-red-200"`)。
