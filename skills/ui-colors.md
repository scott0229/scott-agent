# 色彩與高對比提示規範 (UI Colors)

這是一個講究專業感的專案，色彩的使用必須非常克制，避免五顏六色造成的視覺疲勞。

## 1. 語意化色彩的限制 (Restrained Semantic Coloring)
- 紅色與綠色 (Red/Green) **僅能**保留在實際的財務盈虧、資金進出 (入金/出金)，或是系統層級的重大狀態改變 (如：指派、刪除、結案)。
- 面對一般狀態或無謂的數值變化，請勿隨意套用紅綠色。

## 2. 微妙的高對比提示 (Subtle Emphasis)
- 當需要強調某些嚴重或特殊的文字、標籤狀態時，請使用帶有底色與高對比文字的組合，避免沉重的色塊大面積著色。
- 範例與寫法：使用 `bg-red-50 text-red-600` 或 `bg-blue-50 text-blue-600`。這個設計模式在畫面上既能有效吸引注意，又不會顯得很刺眼。

## 3. 暖色系中性主按鈕 (Warm Neutral Actions)
- 對於 Dashboard 的核心重要操作按鈕 (例如面板上的「新增」主觸發按鈕)，如果想要呈現無壓迫感的高級質感，應考慮採用介於米色與棕色之間的暖中性色調。
- 推薦使用的 Tailwind Class 設定：`bg-[#EAE0D5] hover:bg-[#DBC9BA] text-[#4A3728]`。這比純黑或鮮豔的主題色彩更為耐看，也能賦予系統更高的質感。
