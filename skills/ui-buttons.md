# 操作面板與按鈕樣式規範 (UI Buttons & Action Bars)

為了與系統整體的高級金融面板風格呼應，系統的操作列 (Action Bar) 與按鈕有著特定的排列與色彩美學。

## 1. 暖色調主操作按鈕 (Primary Action Button)
- **視覺呈現**：用於面板最重要的觸發行為，例如位於操作列最右側的「+ 新增」。這類按鈕採用實心的暖色系。
- **樣式代碼**：`className="bg-[#EAE0D5] hover:bg-[#DBC9BA] text-[#4A3728] border-none"`
- **特色**：無邊框、帶有圓角，比起一般的深色或亮藍色，能呈現更無壓迫感的專業氣質。

## 2. 次要與通用操作按鈕 (Secondary / Outline Buttons)
- **視覺呈現**：用於如「寄出報告」、「備份」、「匯入報表」等輔助行為。它們在主操作按鈕的左側一字排開。
- **配置原則**：
  - 使用 `variant="outline"` 屬性。
  - 保留極淺色或透明的背景，加上與背景協調的淡色邊框。
  - 若動作有明確的意義，一律在文字左邊加上相關的圖示 (例如：`<Upload className="mr-2 h-4 w-4" />`)。
- **破壞性次要操作 (Destructive Outline)**：如「刪除全部」，平常必須與其他次要按鈕長得一樣低調，但在 Hover 時才浮現警告色：`className="hover:bg-red-50 hover:text-red-600 hover:border-red-200"`。

## 3. 下拉過濾選單的對齊 (Filter Dropdowns)
- **視覺呈現**：在按鈕列的最左側，經常會搭配像是「所有帳號」的 `Select` 過濾器。
- **樣式要求**：下拉選單的外觀 (高度、邊框顏色、圓角與字體大小) 必須與右側的次要操作按鈕保持絕對的視覺一致性，形成整齊劃一的操作橫排。 

## 4. 避免過多的強調 (Avoid Over-Emphasis)
- 整個操作橫排中，**只能有一個**帶有實心醒目背景色的主按鈕 (通常是在最右側的「+ 新增」或「查詢」)。
- 其餘全部按鈕皆應退居二線使用 Outline，避免畫面上出現多個實心色塊互相爭奪視覺焦點。
