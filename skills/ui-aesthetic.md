# 介面美學設計規範 (UI Aesthetic)

當為此專案進行介面開發時，必須遵循「專業、極簡、純淨」的設計原則，目標是打造無干擾的高級金融面板體驗。

## 1. 極簡優先 (Minimalism)
- 內容優先於裝飾。除非有明確的功能分類需求，否則請不要在標題、頭像或內容區塊加上不必要的圖示 (Icons) 或裝飾性邊框。
- 介面應該專注於資料的呈現，避免過度設計。

## 2. 表單控制項 (Form Controls)
- 不要在這類專業表單的輸入框 (如 `Input`, `SelectTrigger`) 上隨意移除邊框或陰影，這會讓使用者找不到點擊輸入的位置。
- **針對數字輸入的優化**：移除瀏覽器預設的數字上下微調按鈕 (Spinners)。對於金額或大數字的操作，請使用 `type="text"` 而非 `type="number"`，以保持外觀的平整與乾淨，避免滑鼠劃過時產生的視覺干擾。

## 3. 全域捲軸隱藏 (Hidden Scrollbars)
- 原則上整個專案的介面都會隱藏系統預設的捲軸列，以維持畫面俐落度。
- 開發清單或需要滾動的結構時，加入 `no-scrollbar` class 即可隱藏捲軸，但依然能保持原有的滾動功能與順暢感。
