# 排版與數字格式化規範 (UI Typography)

數字與數據呈現是專業金融面板的靈魂，展示數值細節也是所有 UI 中的關鍵任務。

## 1. 數字與百分比精準度 (Percentages & Precision)
- **人類易讀比例**：一般的常見效能比率 (例如：勝率 Win Rate、毛利率等)，請至少穩定保留一個小數點後的精準度 (如顯示為 `85.2%` 而非單純化約成 `85%`)，以便讓閱讀者能快速並精準掃描變化。
- **深度分析專門指標**：對於專精的分析項目，尤其如期權隱含波動率 (IV)、Delta 等可能非常微小卻會造成重大槓桿影響的金融指標，請將轉換公式設定為「保留至少 3 位小數的精準度」 (顯示為 `0.365` 或百分比的 `0.008%`)。這能確實避免有價值的極小數字因為一般轉換被直接四捨五入抹平成無意義的 `0.0%`。

## 2. 大數字與金額排讀性 (Large Values)
- 任何代表大額度的整數型態數據或是資金數目，**絕對必須且強制**透過千分位逗號 (`,` Separator) 進行輔助格式化處理後再渲染至畫面上。
- 通常會建立一個簡單的 `format` 程式區塊，多利用 JavaScript 原生的 `new Intl.NumberFormat('en-US').format(val)` 機制。這能實質地幫助從業者一眼輕易判別與驗證十萬或幾百萬金額之間的位數差別。
